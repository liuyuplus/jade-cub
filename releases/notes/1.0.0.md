## 亮点
- `1.0.0` 是 Jade Cub 的首个公开版本，采用新的产品名、README、DMG 安装背景和 GitHub Release 链路。
- README 首页换成 Jade Cub 专属绿色熊头宣传图，集中展示状态岛、Obsidian 设置和本地 Shelf。
- 设置页默认聚焦 Codex-first 工作流，远程和集成管理入口先隐藏，保留代码以便未来按需求恢复。
- 新增 Obsidian 今日任务配置页，可按用户自己的 vault、Daily 目录、文件名模式和模板路径读取本地任务进度。

## 修复
- 修复设置侧栏选中态在深色背景下文字对比度不足的问题。
- 调整 Codex 思考态三点显示，移除多余气泡背景，并优化闭合刘海中的头像与白点间距。
- 优化网易云等音乐客户端启动/播放后的 Now Playing 探测节奏，减少状态图标延迟感。
- 修复 Codex aborted turn 未及时清理 running task，导致状态可能停留在思考/忙碌的问题。
- 清理公开发布链路中的旧 Ping Island DMG 视觉资源，保留上游署名和 Apache 2.0 许可证链路。

## 说明
- Jade Cub 基于 Ping Island 改造而来，项目保留上游致谢；源码中的 `PingIsland` target 和部分内部 bridge 名称暂时保留以降低重构风险。
- Build 号从上游版本线提升到 `100`，作为 Jade Cub 公开发布的新起点。
- 当前建议从 GitHub Release 下载 Developer ID 签名并 notarized 的 DMG；未签名包仅适合本地测试。

## Highlights
- `1.0.0` is the first public Jade Cub release, with product naming, README links, DMG installer artwork, and GitHub Release URLs moved to Jade Cub.
- The README hero now uses Jade Cub's dedicated green-bear promo image, showing the status island, Obsidian setup, and local Shelf together.
- Settings now default to a focused Codex-first workflow. Remote and integration management controls are hidden for now, while the underlying code remains available for future demand.
- Added an Obsidian daily-task settings page so users can configure their own vault, daily-note directory, filename pattern, and template path.

## Fixes
- Fixed low-contrast selected text in the settings sidebar.
- Tuned the Codex thinking dots by removing the extra bubble background and improving the closed-notch mascot/dots spacing.
- Improved the Now Playing probe cadence after music app launch/playback events, reducing perceived delay for clients such as NetEase Cloud Music.
- Fixed aborted Codex turns so stale running tasks no longer keep the island in a thinking/busy state.
- Removed old Ping Island DMG marketing artwork from the public release surface while preserving upstream attribution and Apache 2.0 license lineage.

## Notes
- Jade Cub is a modified derivative of Ping Island. The `PingIsland` target and some internal bridge names remain for now to avoid unnecessary churn.
- The build number moves to `100` as the start of Jade Cub's public release line.
- For public distribution, prefer Developer ID signed and notarized GitHub Release DMGs. Unsigned packages are for local testing only.
